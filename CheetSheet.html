Cheat Sheet

World Wide Web

  • HTML (HyperText Markup Language): Language used to create web pages.
  • URL (Uniform Resource Locator): Unique address for web resources.
  • HTTP (Hypertext Transfer Protocol): Protocol for data transmission over the web.

URL (Uniform Resource Locator)

The URL is an address used to access resources on the internet.

Syntax:

scheme://host:port/path?query#fragment
  • Scheme: Protocol (e.g., http, https)
  • Host: Domain name (e.g., www.example.com)
  • Port: Optional (e.g., :80)
  • Path: Location of the resource (e.g., /path/to/resource)
  • Query: Optional parameters (starts with ?)
  • Fragment: Optional section reference (starts with #)

Web Servers

A web server delivers web content to clients (browsers) over the internet.

Function:

  • Stores, processes, and delivers web pages to clients upon request.
  • Handles HTTP/HTTPS requests and responses.
  • Executes server-side scripts and delivers dynamic content.

HTML Syntax

Doctype

All HTML documents must start with a DOCTYPE tag.
The tag is not an HTML tag, but rather the tag is more of an information to the browser about the type of document for the browser.
Hence, the name DOCTYPE.

HTML tag

The < HTML > tag is there to take all the elements of other HTML elements.
Except for DOCTYPE tag.

HEAD tag

The < head > tag is used to access metadata (data within data).
MetaData typically define things in an HTML webpage like:

  • Title
  • Style
  • Link and etc.

EM tag

The em tag simply emphasize a text in the HTML file.
eg. The < em >project< /em > has to be completed before Monday.

Strong tag

The strong tag is another way to emphasize a text.
e.g. The < strong >assignment < /strong > has to be given on < strong >Tuesday < /strong >

Comments

Comments in html can be useful if you want to let other developers know some information about the website.
You can also use it to hide content.
eg. < !-- This is a comment -- >..
